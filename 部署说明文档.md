## 📦 项目打包说明

### 一、前端打包

前端已使用 Vite 打包完成，生成的文件位于 `front_end/dist` 目录。

**打包命令**：
```bash
cd front_end
npm run build
```

**打包结果**：
- 打包后的文件位于：`front_end/dist/`
- 包含：HTML、CSS、JavaScript 等静态资源
- 已压缩优化，可直接部署

---

### 二、后端打包

**方式一：使用 Maven 命令打包（推荐）**

```bash
# 在项目根目录执行
mvn clean package -DskipTests
```

打包后会在 `target` 目录生成：
- `cloudnote-0.0.1-SNAPSHOT.jar` - 可执行的 Spring Boot JAR 包

**方式二：使用 IDE 打包**

1. 在 IntelliJ IDEA 中打开项目
2. 右侧 Maven 面板 → Lifecycle → clean
3. 右侧 Maven 面板 → Lifecycle → package
4. 等待打包完成，在 `target` 目录查看生成的 JAR 文件

## 🚀 部署运行说明

### 环境要求

#### 后端环境
- **JDK**：1.8 或以上
- **MySQL**：8.0 或以上
- **Maven**：3.6+ (仅打包时需要)

#### 前端环境
- **Node.js**：16.0 或以上
- **npm**：8.0 或以上

---

### 部署步骤

#### 1. 数据库准备

**创建数据库**：
```sql
CREATE DATABASE cloudnote CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**导入数据库脚本**：
- 执行 `src/main/resources/db/schema.sql`（如果有）
- 或者首次运行时，Spring Boot 会自动创建表结构

**配置数据库连接**：
编辑 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/cloudnote?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root
    password: 你的数据库密码
```

---

#### 2. 后端部署

**方式一：直接运行 JAR 包（生产环境）**

```bash
# 进入 target 目录
cd target

# 运行 JAR 包
java -jar cloudnote-0.0.1-SNAPSHOT.jar
```

**方式二：使用 IDE 运行（开发环境）**

1. 在 IntelliJ IDEA 中打开项目
2. 找到 `src/main/java/com/cloudnote/CloudNoteApplication.java`
3. 右键 → Run 'CloudNoteApplication'

**验证后端启动成功**：
- 访问：http://localhost:8080
- 查看控制台输出，确认无错误

---

#### 3. 前端部署

**方式一：使用 Nginx 部署（生产环境）**

1. 将 `front_end/dist` 目录下的所有文件复制到 Nginx 的 `html` 目录
2. 配置 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root /path/to/front_end/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

3. 重启 Nginx：`nginx -s reload`

**方式二：使用开发服务器（开发环境）**

```bash
cd front_end
npm run dev
```

访问：http://localhost:5173

---

#### 4. 访问系统

**开发环境**：
- 前端：http://localhost:5173
- 后端：http://localhost:8080
- API 文档：http://localhost:8080/doc.html

**生产环境**：
- 系统：http://localhost (或你的域名)
- API 文档：http://localhost/api/doc.html

## 🔧 常见问题解决

### 1. 后端启动失败

**问题**：数据库连接失败
**解决**：
- 检查 MySQL 是否启动
- 检查数据库用户名和密码是否正确
- 检查数据库是否已创建

**问题**：端口 8080 被占用
**解决**：
- 修改 `application.yml` 中的端口号
- 或者关闭占用 8080 端口的程序

### 2. 前端访问失败

**问题**：API 请求失败（跨域）
**解决**：
- 检查后端 CORS 配置
- 检查前端 API 地址配置

**问题**：页面空白
**解决**：
- 检查浏览器控制台错误信息
- 检查前端打包是否成功

### 3. AI 助手无法使用

**问题**：API 调用失败
**解决**：
- 检查 API Key 是否正确
- 检查网络连接（国外服务需要翻墙）
- 尝试切换其他 AI 提供商
