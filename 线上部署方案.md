## ⭐ 方案一：Railway（最推荐，最简单）

### 优点
- ✅ **完全免费**（每月5美元额度，够用）
- ✅ **最简单**（一键部署，5分钟搞定）
- ✅ **支持MySQL**（内置数据库）
- ✅ **自动HTTPS**
- ✅ **免费域名**（xxx.railway.app）
- ✅ **支持Spring Boot + MySQL**

### 费用
- 免费额度：每月 $5
- 你的项目预计使用：$2-3/月
- **完全够用，不需要付费！**

### 部署步骤

#### 1. 注册Railway账号
1. 访问：https://railway.com
2. 使用GitHub账号登录（推荐）
3. 验证邮箱

#### 2. 准备代码
```bash
# 确保项目有这些文件
- pom.xml
- src/
- application.yml
```

#### 3. 部署后端
1. 点击"New Project"
2. 选择"Deploy from GitHub repo"
3. 连接你的GitHub仓库
4. Railway会自动检测Spring Boot项目
5. 自动构建和部署

#### 4. 添加MySQL数据库
1. 在项目中点击"New"
2. 选择"Database" → "MySQL"
3. Railway会自动创建数据库
4. 自动注入环境变量到后端

#### 5. 配置环境变量
Railway会自动设置：
- `MYSQL_URL`
- `MYSQL_USER`
- `MYSQL_PASSWORD`

你需要修改`application.yml`使用这些变量：
```yaml
spring:
  datasource:
    url: ${MYSQL_URL}
    username: ${MYSQL_USER}
    password: ${MYSQL_PASSWORD}
```

#### 6. 部署前端
1. 在同一个项目点击"New"
2. 选择"Empty Service"
3. 上传`dist`文件夹
4. 或者使用Nginx配置

#### 7. 获取域名
- 后端：`https://你的项目名.railway.app`
- 前端：`https://你的前端名.railway.app`

### 预计时间：30分钟

2. 使用GitHub账号登录

#### 2. 部署后端
1. 点击"New +" → "Web Service"
2. 连接GitHub仓库
3. 配置：
   - Name: cloudnote-backend
   - Environment: Java
   - Build Command: `mvn clean package -DskipTests`
   - Start Command: `java -jar target/cloudnote-1.0.0.jar`
   - Instance Type: Free

#### 3. 添加数据库
使用免费MySQL服务：
- **Aiven**（推荐）：https://aiven.io/free-mysql-database

#### 4. 部署前端
1. 点击"New +" → "Static Site"
2. 连接GitHub仓库（front_end目录）
3. Build Command: `npm run build`
4. Publish Directory: `dist`

#### 5. 获取域名
- 后端：`https://cloudnote-backend.onrender.com`
- 前端：`https://cloudnote-frontend.onrender.com`

nohup java -jar cloudnote-1.0.0.jar &

## 📝 Railway 详细部署教程

### 第一步：准备GitHub仓库

1. **创建GitHub仓库**（如果还没有）
```bash
# 在项目目录
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/cloudnote.git
git push -u origin master
```

2. **确保项目结构正确**
```
cloudnote/
├── src/
├── pom.xml
├── front_end/
│   ├── dist/
│   └── package.json
└── README.md
```

### 第二步：修改配置文件

修改 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    # 使用Railway环境变量
    url: jdbc:mysql://${MYSQLHOST:localhost}:${MYSQLPORT:3306}/${MYSQLDATABASE:cloudnote}?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: ${MYSQLUSER:root}
    password: ${MYSQLPASSWORD:123456}
    driver-class-name: com.mysql.cj.jdbc.Driver

server:
  # Railway会自动分配端口
  port: ${PORT:8080}
```

提交更改：
```bash
git add .
git commit -m "Update config for Railway"
git push
```

### 第三步：部署到Railway

1. **登录Railway**
   - 访问：https://railway.com
   - 点击"Login"
   - 选择"Login with GitHub"

2. **创建新项目**
   - 点击"New Project"
   - 选择"Deploy from GitHub repo"
   - 选择你的cloudnote仓库

3. **添加MySQL数据库**
   - 在项目页面点击"New"
   - 选择"Database"
   - 选择"Add MySQL"
   - Railway会自动创建数据库

4. **连接数据库到后端**
   - Railway会自动注入环境变量
   - 变量名：MYSQLHOST, MYSQLPORT, MYSQLDATABASE, MYSQLUSER, MYSQLPASSWORD

5. **导入数据库**
   - 点击MySQL服务
   - 点击"Connect"
   - 使用提供的连接信息
   - 导入你的cloudnote.sql文件

6. **部署前端**
   - 在项目中点击"New"
   - 选择"Empty Service"
   - 添加Nginx配置
   - 或使用Vercel部署前端（更简单）

7. **获取域名**
   - 点击后端服务
   - 点击"Settings" → "Networking"
   - 点击"Generate Domain"
   - 获得：https://你的项目名.railway.app

### 第四步：配置前端

修改前端API地址：

`front_end/src/utils/request.js`：
```javascript
const request = axios.create({
  baseURL: 'https://你的项目名.railway.app', // Railway后端地址
  timeout: 10000
})
```

重新打包前端：
```bash
cd front_end
npm run build
```

### 第五步：测试访问

1. 访问后端：https://你的项目名.railway.app
2. 访问前端：部署到Vercel或Railway
3. 测试功能

## 💡 常见问题

### Q1：Railway免费额度够用吗？
A：够用！你的项目预计使用$2-3/月，免费额度是$5/月。

### Q2：域名可以自定义吗？
A：可以！Railway支持自定义域名，但需要你自己购买域名。

### Q3：数据会丢失吗？
A：不会！Railway的MySQL是持久化存储。

### Q4：可以用多久？
A：只要在免费额度内，可以一直用。

### Q5：如果超出免费额度怎么办？
A：Railway会暂停服务，不会扣费。你可以升级到付费计划。

### Q6：部署失败怎么办？
A：查看Railway的日志，通常是配置问题。
